# Reasoning Param Injector

SillyTavern(ST)에서 **Custom(OpenAI 호환) 엔드포인트**를 쓸 때, 모델별로
`reasoning_effort` / `verbosity` 값을 저장해두고 요청 직전에 자동으로 주입해 주는
확장입니다.

> ST가 보내주지 않는 추론 파라미터를, **모델별로** 관리하며
> 요청에 끼워 넣어 줍니다.

---

## Background Info

### 1) ST의 추론 파라미터 주입 방식

ST의 프리셋에는 `Request model reasoning` 설정이 있어 `reasoning_effort`
(none/minimum/low/medium/high/maximum)와 `verbosity`를 고를 수 있습니다.
ST는 이 값을 **모든 모델에 보내지 않습니다.** 내부에 "이 모델은 추론
파라미터를 받는다"는 **화이트리스트**(OpenAI 계열 모델 목록)를 두고, 거기에
**정확히 일치하는 모델명일 때만** `reasoning_effort`를 상위로 전송합니다.

즉 화이트리스트에 없는 모델은, 프리셋에서 추론 값을 골라도 **그 값이 아예
전송되지 않습니다.**

### 2) 화이트리스트에 관한 ST 공식 정책

- 이 목록은 **OpenAI 계열 모델 전용**입니다(`o1`, `o3`, `gpt-5` 계열 등).
- 따라서 **Claude·Gemini 등은 정식 모델명이어도 목록에 없습니다.**  그 경우 ST는 추론 파라미터를 전송하지 않습니다. (Velbosity의 경우 전체 일치성을 보지 않고 정식 모델명으로 시작하는 경우 통과됨)

### 3) 미통과 모델의 경우

화이트리스트 미통과 모델이라도, ST의 **추가 파라미터(Additional Parameters,
`custom_include_body`)** 에 직접 `reasoning_effort: high` 같은 값을 써넣으면
화이트리스트를 우회해 요청에 실립니다.

그러나 ST는 이 추가 파라미터를 **"Custom 공급자" 전체에서 공통으로 하나만**
관리합니다. 서로 다른 Custom 엔드포인트를
오갈 때마다, 또는 모델을 바꿀 때마다 추가 파라미터 칸을 일일이 고쳐 써야 합니다.

**확장의 역할:**
추론 파라미터를 **모델명 단위로 따로 저장**해 두고, 현재 연결된 모델에 맞는 값을
요청 직전에 자동으로 주입합니다. 더 이상 엔드포인트/모델을 바꿀 때마다 추가
파라미터를 수정할 필요가 없습니다.

---

## 동작 방식

요청 직전(ST의 `CHAT_COMPLETION_SETTINGS_READY` 시점)에, 현재 활성 모델명을
화이트리스트와 **자동 대조**합니다.

- **화이트리스트 통과 모델** → ST가 알아서 정식 전송하므로 **주입하지 않습니다.**
  (  이 경우 프리셋 메뉴에서 직접 조절하세요.)
- **미통과 모델** → 추가 파라미터 경로(`custom_include_body`)로 저장값을
  **주입합니다.** 이는 추가 파라미터 칸에 직접 써넣은 것과 동일하게 동작합니다.
- `reasoning_effort`와 `verbosity`는 **각각 따로** 판독·주입합니다.

> **유의사항**
> 이 확장은 **ST 단에서 파라미터를 요청에 실어 보내는 것까지만** 보장합니다.
> 그 값을 받은 **서버가 실제 모델에 전달할지는 보장하지
> 않습니다.** 서버는 자신이 지원하지 않는 파라미터를 버리거나
> 무시할 수 있습니다. 따라서 "주입함" 로그가 떠도, 그 값이 모델까지 도달했는지는 **별개의
> 문제**이며, 사용자가 직접 확인·실험해야 합니다.

### Reasoning Effort 값

드롭다운은 실제 전송값을 그대로 보냅니다: `none / minimum / low / medium / high /
maximum`, 그리고 OpenAI 계열 전용인 `minimal / xhigh`. `none`은 추론을 끄는
값입니다. 모델이 받지 않는 값이면 서버가 거부할 수 있으니, 거부되면 다른 값을
고르거나 커스텀 파라미터로 정확한 값을 보내세요.

### 자동매칭 (기본 켜짐)

모델명으로 회사를 추정해, reasoning_effort를 그 회사가 받는 값으로 보정합니다.

- **none**: 변환하지 않고 그대로 전송(추론 끄기 의미 보존)
- **OpenAI**(`gpt`/`o1`/`o3`/`o4`): `minimum→minimal`, `maximum→xhigh`
- **DeepSeek**(`deepseek`): `maximum/xhigh→max`, `나머지→high`
- **Claude**(`claude`): `minimum/minimal→low`, `maximum→max`
- **그 외(Gemini 등)**: 변환 없이 raw 전송 (전송 형식이 불확실해 보류)

매칭 결과는 활성 모델 박스에 작게 표시됩니다. 끄면 모든 값을 raw로 보냅니다.

### 추가할 파라미터 / 제외할 파라미터 (각 최대 5개)

reasoning/verbosity 외의 파라미터를 직접 제어합니다(접힘 메뉴). 모델명 기준 저장.

- **추가할 파라미터**(키·값): 임의 파라미터를 요청에 주입합니다. 자동매칭 없이
  입력한 그대로 전송. 예: 키 `thinkingLevel`, 값 `high`.
- **제외할 파라미터**(키만): ST가 보내는 요청에서 그 키를 제외합니다(최종 body에서
  삭제). 예: `frequency_penalty`.

이 두 가지는 **화이트리스트 통과 모델에도 항상 적용**됩니다(유저 자유 영역).
ST가 이미 보내는 값과 충돌할 수 있으니, 그 부분은 사용자가 판단해 사용하세요.

## 적용 범위

- **Custom(OpenAI 호환) 소스에서만 동작합니다.** Vertex/Gemini/Claude 공식 등
  다른 연결 소스는 ST 내부에서 요청 경로 자체가 달라(`custom_include_body`를
  사용하지 않음), 이 확장의 주입이 적용되지 않습니다. 그런 소스에서는 패널에
  "이 소스는 미지원"으로 표시되며, 추론 값은 프리셋에서 조절해야 합니다.

---

## 사용법

1. 확장을 설치하고 ST를 재시작합니다.
2. Extensions 패널의 **Reasoning Param Injector** 드로어를 엽니다.
3. **현재 연결**에 지금 활성 모델명과 판독 결과(주입 대상 / ST 자체 전송 / 미지원)가
   자동 표시됩니다.
4. **Reasoning Effort / Verbosity** 값을 고르고 **이 모델에 저장**을 누릅니다.
   이후 같은 모델로 요청하면 저장값이 자동 주입됩니다.
5. **로그**에서 주입/스킵 내역을 확인할 수 있습니다(화면 표시는 토글로 켜고 끔).

> **결과값 확인은 스트리밍을 끄고(non-stream) 하세요.** 스트리밍을 켜면 응답의
> usage(토큰 상세)가 누락되거나 확인하기 어려워, 추론 토큰 등을 비교하기
> 곤란합니다.

## 화이트리스트 자동 갱신

- 패널의 **GitHub에서 자동 갱신** 버튼은 ST 공식 저장소에서 최신 화이트리스트를
  받아옵니다(텍스트만 파싱하며, 코드를 실행하지 않습니다).
- 갱신에 실패하면 확장에 내장된 목록을 그대로 사용합니다.
- 목록에 없는 모델을 직접 넣고 싶으면 **수동 추가 모델** 칸에 한 줄씩 정확한
  모델명을 입력하세요.

## 저장 위치 / 안전성

- 데이터는 확장 전용 **IndexedDB**(`st_param_injector`)에 저장됩니다.
  모델별 설정은 `models` 스토어에 **모델명**을 키로 저장됩니다.
- **저장된 모든 모델 설정 삭제** 버튼으로 한 번에 비울 수 있습니다.
